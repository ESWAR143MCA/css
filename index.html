1) Fundamentals: Syntax & Selectors

Know how CSS targets elements and resolves conflicts.

Cheat Sheet
  • Syntax: selector { property: value; }
  • Selector Types: universal *, type div, class .btn, id #app, attribute [data-x], pseudo‑class :hover, pseudo‑element ::before
  • Combinators: descendant div p, child ul > li, adjacent h1 + p, general sibling h1 ~ p
  • Specificity: inline > id > class/attr/pseudo‑class > type; later wins on tie.
Examples 
/* Attribute, pseudo-class, and combinator example */
input[type="email"]:focus + .hint { opacity: 1; }

/* Specificity demo */
#hero .btn.primary { background: royalblue; }
/* overrides */
.btn.primary { background: steelblue; }

2) Box Model

Master content vs padding vs border vs margin. Prefer box-sizing: border-box;.

Diagram & Tips

Set *{box-sizing:border-box} to make width/height include padding+border. Use overflow to control scroll/clip; avoid margin collapse surprises by using padding on parent containers.

Example 
*{ box-sizing: border-box; }
.card{ width: 300px; padding: 16px; border: 2px solid #1f2937; }

3) Layout: Display, Flow & Positioning

Choose the right layout mode: normal flow, flexbox, grid, or positioning.

Position
  • static (default), relative (offset, anchor for absolute), absolute (removed from flow), fixed, sticky.
Flexbox — 1D alignment 
.row{
  display:flex;
  gap:12px;
  justify-content:space-between; /* main axis */
  align-items:center;            /* cross axis */
  flex-wrap:wrap;
}
.col{ flex: 1 1 200px; }
Grid — 2D control 
.grid{
  display:grid;
  grid-template-columns: repeat(auto-fit, minmax(220px, 1fr));
  gap: 14px;
}
/* Named areas */
.layout{
  display:grid;
  grid-template:
    "head head" auto
    "side main" 1fr / 260px 1fr;
}

4) Typography

Key Rules
  • Use system stacks or variable fonts to reduce layout shift.
  • Comfortable reading: line-height: 1.5–1.8, 60–75 chars/line.
  • Use rem for scalable type; reserve em for component-relative sizing.
Snippet 
:root{ font-size: 16px; }
h1{ font-size: clamp(1.75rem, 3vw, 2.25rem); }
p{ line-height: 1.65; letter-spacing: .2px; }

5) Color, Variables & Theming

Use CSS variables for themes; ensure contrast (WCAG AA+).

Variables + Dark Mode 
:root{ --brand: #2563eb; --bg: #ffffff; --text:#0b1220; }
@media (prefers-color-scheme: dark){
  :root{ --bg:#0b1220; --text:#e5e7eb; }
}
body{ background: var(--bg); color: var(--text); }

6) Responsive Design

Modern Approach
  • Use fluid sizing with clamp() and flexible layouts.
  • Media queries when necessary; prefer content‑driven container queries.
Examples 
.card{ padding: clamp(12px, 2vw, 24px); }
@container (min-width: 500px){
  .tile{ grid-template-columns: repeat(3,1fr); }
}
@media (max-width: 640px){
  nav { display:none; }
}

7) Transitions & Animations

Smooth Interactions 
.btn{ transition: transform .2s ease, box-shadow .2s ease; }
.btn:hover{ transform: translateY(-1px); box-shadow: 0 6px 20px rgba(0,0,0,.25); }

@keyframes pulse{ from{ scale:1 } 50%{ scale:1.04 } to{ scale:1 } }
.badge{ animation: pulse 2s infinite }

8) Best Practices & Architecture

Guidelines
  • Scope styles by component (BEM or utility-first).
  • Minimize deep selectors; prefer single‑class hooks.
  • Use logical properties (margin-inline, padding-block) for RTL support.
  • Measure: DevTools Performance, Coverage.

9) Cascade Layers & Specificity Control

@layer + Reset 
@layer reset, base, components, utilities;

@layer reset{
  *,*::before,*::after{ box-sizing:border-box; }
  body{ margin:0 }
}
@layer components{
  .btn{ padding:.6rem 1rem; border-radius:.6rem }
}
@layer utilities{
  .mt-2{ margin-top:.5rem }
}

10) Misc Essentials

Quick Hits
  • Units: rem (root), em (relative), vh/vw (viewport), fr (grid).
  • Images: object-fit, aspect-ratio, border-radius, clip-path.
  • Stacking: new stacking context via position + z-index, opacity, transform, filter.
Snippet 
.avatar{
  width:100px; aspect-ratio:1; object-fit:cover; border-radius:50%;
  overflow:hidden;
}
